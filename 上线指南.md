# 个人网站上线指南

> 时间：2026-07-02 ~ 2026-07-03  
> 域名：attention-evolution.com  
> 技术栈：Astro 4 + Tailwind CSS + GitHub Pages

---

## 流程概览

```
写代码（本地 Mac）
    │
    ▼
git push → GitHub（代码仓库）
    │
    ▼
GitHub Actions → 自动构建 Astro → 部署到 GitHub Pages
    │
    ▼
阿里云 DNS → 把域名指向 GitHub Pages
    │
    ▼
用户访问 attention-evolution.com → 看到网站
```

## 第一步：准备域名

**去哪里买**：阿里云万网（wanwang.aliyun.com）、腾讯云 DNSPod 都行

**注意**：
- `.com` 域名在国内注册商处购买后，需要**实名认证**才能激活
- 实名认证通过前，域名会被标记 `clientHold`，DNS 不生效
- 审核通常 1-2 个工作日

**本次经验**：在阿里云万网买了 `attention-evolution.com`，提交实名认证后状态从 `clientHold` 变为正常。

## 第二步：代码推到 GitHub

### 2.1 初始化 git

```bash
git init
git add .
git commit -m "初始化网站"
```

### 2.2 创建 GitHub 仓库

1. 打开 github.com → 右上角 + → New repository
2. Repository name 任意（本次用了 `attention-evolution`）
3. 选 **Public**
4. ❌ 不要勾选 README、.gitignore、License（保持空仓库）

### 2.3 配置 SSH 密钥（一次性操作）

**注意**：国内网络环境下，SSH 直连 GitHub 可能被拦截。如果遇到 `Connection closed` 错误，不是密钥问题，是网络问题。

```bash
# 生成密钥
ssh-keygen -t ed25519 -C "你的邮箱@example.com" -f ~/.ssh/id_ed25519 -N ""

# 复制公钥
cat ~/.ssh/id_ed25519.pub
```

然后在 github.com → Settings → SSH and GPG keys → New SSH key，粘贴公钥。

**重要**：第一次连接需要手动接受 GitHub 的 host key：
```bash
ssh-keyscan -t ed25519 github.com >> ~/.ssh/known_hosts
```

### 2.4 推送代码

```bash
git remote add origin git@github.com:你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

## 第三步：部署到 GitHub Pages

### 3.1 创建 GitHub Actions 工作流

在项目根目录创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: withastro/action@v2

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### 3.2 开启 GitHub Pages

1. 打开仓库 Settings → Pages
2. **Source** 选 **GitHub Actions**（不是 Deploy from a branch）
3. 每次 `git push` 后 GitHub 会自动构建和部署

### 3.3 关于 Cloudflare Pages（踩坑记录）

**⚠️ 不要用**。本次尝试了 Cloudflare Pages，遇到两个问题：
- Cloudflare 自动检测到 Astro 框架后，会运行 `astro add cloudflare`，把静态站点改成了需要 Worker 的 SSR 模式
- 提示 `You need to register a workers.dev subdomain`，需要额外开通 Workers 服务

**结论**：纯静态网站直接走 GitHub Pages，不用 Cloudflare。

## 第四步：绑自定义域名

### 4.1 astro.config.mjs 配置

```js
export default defineConfig({
  site: 'https://attention-evolution.com',  // 改为自己的域名
  // ...
});
```

### 4.2 GitHub Pages 添加自定义域名

仓库 Settings → Pages → Custom domain → 填入 `attention-evolution.com` → Save

GitHub 会自动创建 CNAME 文件，并尝试验证 DNS。

### 4.3 阿里云 DNS 配置

需要添加 **5 条记录**：

| 记录类型 | 主机记录 | 记录值 |
|---|---|---|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |
| CNAME | www | `你的用户名.github.io` |

**为什么要 4 条 A 记录**：GitHub Pages 有 4 台全球服务器做负载均衡和灾备，4 条同时配更稳定。

**⚠️ 关键：`www` 子域名不能漏**

本次踩过最大的坑：只加了 4 条 `@` 的 A 记录，漏了 `www` 的 CNAME 记录。

- `attention-evolution.com` ✅ 有记录
- `www.attention-evolution.com` ❌ 没有记录 → GitHub 校验失败

**错误提示**：
```
www.attention-evolution.com is improperly configured
Domain's DNS record could not be retrieved.
```

**解决**：补一条 CNAME 记录，主机记录填 `www`，记录值填 `luyoyo119.github.io`。

### 4.4 开启 HTTPS

绑定成功后，GitHub Pages 自动签发 Let's Encrypt 证书（3-5 分钟）。

页面上会出现 **Enforce HTTPS** 勾选框，勾上。

## 第五步：验证

### 5.1 检查 DNS 是否生效

如果本机 DNS 被拦截（国内常见），使用 DNS-over-HTTPS 验证：

```bash
curl -s -H "accept: application/dns-json" \
  "https://1.1.1.1/dns-query?name=attention-evolution.com&type=A"
```

应返回 4 个 GitHub IP（185.199.108/109/110/111.153）。

### 5.2 检查站点

```bash
curl -s -o /dev/null -w "%{http_code}" https://attention-evolution.com
# 应返回 200
```

## 本次遇到的所有问题

| 问题 | 原因 | 解决 |
|---|---|---|
| 域名 `clientHold` | 未实名认证 | 阿里云提交实名 |
| SSH `Connection closed` | 没接受 GitHub host key | `ssh-keyscan github.com >> ~/.ssh/known_hosts` |
| Cloudflare Pages 部署失败 | 自动装了 Cloudflare Worker 适配器 | 换成 GitHub Pages |
| GitHub Pages 报 `InvalidDNSError` | 漏了 `www` 子域名 CNAME 记录 | 补加 CNAME |
| DNS 查询返回假的 198.18.0.x | 本机 DNS 被网络环境拦截 | 用 DNS-over-HTTPS 验证 |
